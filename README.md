# Metodología OWISAM-HS
## Análisis de Hotspots Inseguros

### Descripción del Proyecto

Este proyecto tiene como objetivo analizar vulnerabilidades presentes en hotspots WiFi abiertos mediante la simulación controlada de ataques en un entorno de laboratorio. Se evaluará el impacto de configuraciones inseguras en redes públicas y se propondrán medidas de mitigación.

### Problema

Las redes WiFi públicas suelen carecer de cifrado y mecanismos avanzados de protección, lo que las convierte en un objetivo frecuente para ataques como:

- Rogue DHCP
- Man-in-the-Middle (MITM)
- Evil Twin
- Manipulación de DNS

Este proyecto busca demostrar experimentalmente estos riesgos en un entorno controlado.

### Objetivos

- Implementar un hotspot abierto en laboratorio.
- Simular un servidor DHCP malicioso.
- Analizar el impacto en dispositivos conectados.
- Evaluar la exposición de tráfico no cifrado.
- Proponer mecanismos de mitigación.

### Herramienta a Desarrollar

Se desarrollará un entorno experimental que permitirá:

- Detectar vulnerabilidades en redes abiertas.
- Analizar tráfico de red.
- Medir el tiempo de compromiso de dispositivos.
- Generar evidencia técnica para análisis académico.

### Metodología

1. Configuración de entorno controlado.
2. Implementación de hotspot abierto.
3. Simulación de ataque.
4. Captura y análisis de tráfico.
5. Evaluación de impacto.
6. Aplicación de medidas defensivas.

### Entorno de Pruebas

- Kali Linux
- Wireshark
- Red WiFi abierta en laboratorio aislado
- Máquina víctima virtualizada

### Alcance

El proyecto se realizará exclusivamente en un entorno controlado y no afectará redes reales o públicas.
